# Week 6 Report

## What I learned and did in **ACIT 2811 UX/UI** last week

### Prototyping

This week focused on understanding how to plan and visualize a design before building it.

### User Flows

User flows show the steps a user takes to complete a task in an app or website.

- Focus on the user’s journey  
- Helps identify how users move between pages  
- Useful for improving usability  

### User Flow Diagrams

User flow diagrams are visual representations of how users navigate through a system.

- Use shapes and arrows to show steps  
- Show decisions and different paths  
- Help understand user behavior

### Low-Fidelity Mockups

Low-fidelity mockups are basic visual representations of a design.

- Simple and quick to create  
- Usually black and white  
- Focus on structure and functionality  
- Not detailed or styled  

### What we got out of class

- Learned how to plan designs before starting development  
- Understood how users interact with an application step by step
