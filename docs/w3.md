## Group Activity: Personas

*A persona is a description of a possible/ideal user
of a product/service. We create personas:*
*  To build empathy
* To identify user characteristics
* To identify user behaviours
* To identify user *goals*

Group Activity: User Stories

**TASK**: Personas identify user characteristics, behaviours, and goals
User Stories identify features (technical features of a product/service that need to be built)

User Stories make up a list of tasks that you can use to ensure, at the end of the project, that you have built everything you set out to build. For each of your two personas, create three User Stories.

**Structure:** "As a _________, I want to _________, so that I can ________"

**Example:** "As a [user], I want to [perform an action], so that I can [achieve a goal]"

As a prospective student, I want to learn about the admission process, so 
that I can start a program

* These need to be written from the perspective of your Personas *
