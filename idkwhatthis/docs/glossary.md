# Glossary

**Term**
**Definition**                                                                                                     
**Git**                   A distributed version control system used to track changes in code and collaborate with others.                       
**GitHub**                A cloud-based platform that hosts Git repositories and enables collaboration, version control, and project management.
**Repository (Repo)**     A storage location for a project’s files and version history, managed using Git.                                      
**Clone**                 The process of creating a local copy of a remote GitHub repository on your machine.                                   
**Commit**                A snapshot of changes made to files in a repository, saved with a message describing the changes.                     
**Branch**                A separate line of development that allows you to work on features or fixes without affecting the main codebase.      
**Main Branch**           The primary branch of a repository where stable code is typically stored (often named `main` or `master`).            
**Remote**                A version of your repository hosted on a platform like GitHub.                                                        
**Origin**                The default name given to the remote repository when linking a local repo to GitHub.                                  
**Push**                  The process of uploading local commits to a remote repository on GitHub.                                              
**Pull**                  The process of fetching and merging changes from a remote repository into your local repository.                      
| **Stage (Staging Area)**  The step where changes are prepared before being committed using `git add`.                                           
**Initialize (git init)** The process of creating a new Git repository in a local project folder.                                               
**Terminal**              A command-line interface used to interact with your system and run Git commands.                                      
**Git Bash**              A terminal application for Windows that provides Git command-line features.                                           
**README**                A file that describes a project, typically including its purpose, setup instructions, and usage.                      
**.gitignore**            A file that specifies which files or folders Git should ignore and not track.                                         
**URL (Repository URL)**  The web address used to access or connect to a GitHub repository.                                                     
**ZIP Download**          A method of downloading a repository as a compressed file without version control features.                           
**Version Control**       A system that records changes to files over time, allowing you to track history and collaborate efficiently.
