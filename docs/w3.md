## Group Activity: Personas

*A persona is a description of a possible/ideal user of a product/service. We create personas:*

- To build empathy  
- To identify user characteristics  
- To identify user behaviours  
- To identify user goals  

### Create two personas (1 page each)
- Photo  
- Name  
- Age  
- Occupation  
- Needs & Goals  
- Pain Points (difficulties)  

---

## Group Activity: User Stories

**TASK:**  
Personas identify user characteristics, behaviours, and goals  
User Stories identify features (technical features of a product/service that need to be built)

User Stories make up a list of tasks to ensure that, at the end of the project, everything is built.

For each of your two personas, create three User Stories.

### Structure
 "As a ___, I want to ___, so that I can ___"

### Example
 As a prospective student, I want to learn about the admission process, so that I can start a program

- These must be written from the perspective of your Personas  

---

## Group Activity: User Scenarios

A User Scenario is a technical description of the steps required to complete one of your user stories  

### USER STORY
As a medical director, I want to download spec sheets so that I can verify compliance details  

### USER SCENARIO
1. Start at the home page  
2. Click the downloads link in the menu  
3. On the downloads page, filter documents  
4. Click the file link
